# ReportAssistant

ReportAssistant 是一个面向开发者的年度量化报告原型，支持选择代码管理服务器（GitLab / SVN）、配置服务器地址并填写账号信息后自动拉取数据，生成提交画像与可视化指标，适合用于展示研发效能数据的产品雏形。

## 设计目标

- **统一入口**：在一个页面完成服务器选择、年度筛选、账号授权与报告生成。
- **可视化驱动**：以卡片、条形、标签、表格等形式展示核心指标，便于快速理解。
- **可扩展数据源**：前端仅模拟拉取流程，后续可替换为真实的 GitLab API / SVN 日志解析。
- **结果可解释**：指标明确说明来源（提交次数、活跃天数、项目数、代码增删、关键词、时间段等）。

## 功能概览

- **服务器选择与地址配置**：支持 GitLab 与 SVN，并允许输入企业内部服务器地址。
- **年度筛选**：选择需要生成报告的年度。
- **账号输入**：用户名 + Token/密码，用于触发自动拉取数据。
- **核心指标**：提交次数、活跃天数、涉及项目数、最大单日提交数。
- **代码增删**：新增/删除行数分布，并给出活跃时间段标签（夜猫子/早鸟）。
- **高频关键词**：基于 Commit message 的高频词展示。
- **偏好项目/模块**：统计项目贡献排名与占比。
- **提交类型分布**：修复型 vs 新功能型提交占比。

## 使用方式

1. 打开项目根目录，启动本地静态服务：

   ```bash
   python -m http.server 8000
   ```

2. 在浏览器访问：

   ```text
   http://localhost:8000
   ```

3. 填写页面表单：
   - 选择 GitLab 或 SVN。
   - 输入服务器地址（如 `https://git.example.com`）。
   - 选择年度。
   - 输入用户名与 Token/密码。
4. 点击 **生成年度报告**，系统会模拟拉取数据并生成可视化报告。

## 系统设计说明

### 页面结构

- `index.html`：定义页面骨架，包括表单、指标卡片、增删图表、关键词标签、项目表格、提交类型分布等模块。

### 视觉样式

- `styles.css`：定义卡片式布局、响应式网格、条形图、标签、表格与状态提示等统一视觉风格，确保移动端也能适配。

### 数据逻辑

- `app.js`：
  - 维护 `mockCommits` 作为演示数据源。
  - `simulateFetchCommits` 模拟真实拉取流程，并在点击生成按钮时触发。
  - `computeMetrics` 计算核心指标、增删行数、关键词统计、提交类型比例等。
  - `renderMetrics` 负责将数据渲染至页面。
